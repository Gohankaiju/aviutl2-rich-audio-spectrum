# rich-audio-spectrum パラメータリファレンス

## 主要パラメータ

どのプリセットを選んでいても直接適用される(プリセットは表示タイプ・配色・
効果・反応カーブの組み合わせを決める)。

| 項目 | 範囲(単位) | 既定 | 説明 |
| --- | --- | --- | --- |
| プリセット | 1-12 | AE Clean Spectrum | README参照 |
| 最小範囲Hz / 最大範囲Hz | 0-22050 / 20-22050 | 0 / 1380 | 表示する周波数範囲(線形軸) |
| 分割数 | 1-2048 | 128 | バー/点の本数。波形系では時間方向の分割数 |
| 横幅 | 100-4000 (px) | 1024 | ステレオスコープでは正方形サイズ |
| 線太さ | 0.5-1024 (px, 0.5) | 5 | バーはスロット幅+1でクランプ |
| 波形高さ | 1-16 (倍率, 0.1) | 4 | 表示px = 値/400000スケール × 倍率 × 画面高。オブジェクト枠は画面高で固定 |
| 上下反転 / ミラー | ON/OFF | OFF | ミラー=下側の追加描画 |
| 弾み% | 0-150 (%) | 0=OFF | バネ慣性(減衰比 1.0→0.03)。ヒットでオーバーシュートし揺り戻す。揺れの速さはAttack msに連動。100超は残揺れ数秒の低減衰域 |
| 落下% | 0-100 (%) | 0=OFF | 下降を重力落下化。全高を0.15秒(低%)〜1.6秒(高%)で落ち切る |
| 色1 / 色2 | 色 | 0x00d9ff / 0xff40ff | 常に適用される配色(プリセットは形状・効果・反応のみを決める) |
| 残像% | -1-100 (%) | -1=プリセット値 | 0=OFF、1-100で層数と減衰を連続制御 |
| 個別オブジェクト | ON/OFF | OFF | バー/点を独立オブジェクト化し後段効果をバー単位適用(上限256本、内蔵残像/SSAA無効) |

## 詳細調整(折りたたみグループ)

| 項目 | 範囲(単位) | 既定 | 説明 |
| --- | --- | --- | --- |
| スケール% | 10-400 (%) | 100 | サイズ系パラメータの一括拡縮 |
| 感度dB | -24-24 (dB, 0.1) | 0 | 入力レベル補正 |
| 反応の深さ% | 10-300 (%) | 100 | 表示振幅の倍率 |
| Attack ms | 0-500 (ms) | 0=プリセット値 | 立ち上がりの時定数。弾みの揺れ速度も決める |
| Release ms | 0-2000 (ms) | 0=プリセット値 | 減衰の時定数(指数)。物体的な落下は落下%で |
| 滑らかさ | -1-100 (%) | -1=プリセット値 | 追加のガウシアン平滑化。0でも自動アンチエイリアスは有効 |
| 品質 | Auto/Preview/Balanced/High/Export | Auto | Auto=編集中Balanced、動画出力中Export(SSAA 2x) |

## 設計方針

- 透明度・座標・拡大率・回転など、AviUtl2標準の描画設定やアニメーション効果で
  実現できるものはUIに持たない。rich-audio-spectrumが持つのは「音への反応の質」だけ。
- 効果量は段階選択ではなく連続量(%)。プリセット上書き系は
  「0または-1=プリセット値」の規約。

## 解析パイプライン(スペクトラム系)

1. `obj.getaudio(..., "fourier")` — 1024bin固定、`bin_hz(i) = rate × i / 2048`
2. 相対ノイズゲート(そのフレームの最大値の10%未満を0に)、DC bin除去
3. 指定範囲の切り出し + 端部パディング
4. ガウシアン平滑化(σ2.5、端ゼロ埋め)
5. エルミート(Catmull-Rom)補間で分割数へリサンプル + 端フェード
6. リニア振幅表示(値 ÷ 400000スケール × 波形高さ × 画面高)
7. 感度 → 反応の深さ → Attack/Release → 弾み → 落下 → 滑らかさ

## 検知分解能の限界(AviUtl2 API制約)

- 周波数方向: `fourier` は1024bin固定。1binの幅は `サンプルレート ÷ 2048`
  (44.1kHzで約21.5Hz)。低域では分割数を増やしても複数バンドが同じbinを共有する。
- 時間方向: 音声取得は1フレーム単位(60fpsで16.7ms毎)。フレーム内の時間変化は取得不可。
- `pcm` の実返却数は1フレーム分(`レート÷fps`)が上限。
